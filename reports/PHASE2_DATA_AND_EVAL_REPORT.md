# Отчёт по Этапу 2: Спецификация, базовая модель, датасет и evaluation

## 1. Спецификация проекта
- Подготовлен `docs/PROJECT_SPEC.md` с требованиями к модели (MVP: только SELECT запросы для SQLite, русский язык).
- Разработана JSON Schema для примеров `schemas/example_record.schema.json`.

## 2. Выбор базовой модели (Research)
Проведён поиск кандидатов в диапазоне 3-4B параметров для QLoRA на RTX 5070 Ti (16 GB VRAM).
**Рекомендация:**
- **Основной кандидат:** `Qwen/Qwen3-4B`
  - Отличный русский язык (pre-training на 119+ языках).
  - Подходит под VRAM (около 7-9 ГБ в 4-bit).
  - Apache-2.0 лицензия (позволяет публикацию деривативов).
  - Thinking Mode (Chain-of-Thought) будет полезен для генерации SQL.
- **Резервный кандидат:** `google/gemma-3-4b-it` (128K контекст, 4.3B, Gemma Terms of Use).
**Исключены:** `Phi-4-mini` (русский официально не поддерживается) и `Llama-3.2-3B` (аналогично).

## 3. Подготовка seed-датасета
- Реализован скрипт генерации `src/rudataanalyst_sql/data/build_seed_dataset.py`, создающий 3 SQLite базы (shop, hr, support).
- Сгенерировано 62 примера, покрывающих различные SQL-концепции (JOIN, GROUP BY, WHERE, LIKE и др.).
- Распределение по сплитам:
  - `train`: 40 примеров (базы `shop` и `hr`).
  - `validation`: 8 примеров (отложенные вопросы для `shop` и `hr`).
  - `test`: 14 примеров (только база `support`, для проверки zero-shot transfer на новые схемы).
- Данные экспортируются в формат HuggingFace Dataset (`data/hf_dataset`).

## 4. Качество данных и Evaluation
- Скрипт `validate_dataset.py` проверяет все JSONL файлы на соответствие схеме и тестирует исполнение SQL.
- Скрипт `leakage_check.py` контролирует отсутствие утечек вопросов или SQL-запросов между сплитами, а также изолированность `test` базы `support`.
- Внедрён модуль `evaluation/sql_executor.py` со строгой защитой от деструктивных SQL (INSERT/UPDATE/DELETE/DROP).
- Развернут пайплайн `make phase2`, который генерирует, валидирует, экспортирует датасет и прогоняет юнит-тесты `pytest`.

Этап 2 завершён успешно. Датасет, метрики и инфраструктура валидации готовы к началу Этапа 3 (Qwen3-4B QLoRA pipeline).
