# Отчет по фазе 3: Базовый инференс и проверка QLoRA (Smoke Test)

## 1. Анализ модели
- **Модель**: `Qwen/Qwen2.5-3B-Instruct`
- **Revision**: `main`
- **Лицензия**: `Apache-2.0`
- **Размер загрузки (кэш)**: ~6.5 ГБ (из них исключены GGUF).

## 2. NF4 Check & GPU Metrics
Проверка на NVIDIA GeForce RTX 5070 Ti (sm_120) показала, что квантование bitsandbytes **NF4** (4-bit) с compute dtype **bfloat16** успешно работает как на этапе forward, так и на этапе backward (при использовании PEFT Lora).
- Пиковое потребление VRAM при инференсе: **2.03 GB**.

## 3. Baseline Results (Zero-shot)
Инференс `test` выборки (14 примеров):
- **Exact Match Accuracy**: 7.14% (1 / 14)
- **Execution Match Accuracy**: 50.00% (7 / 14)
- **Execution Error**: 1 (Синтаксическая ошибка или неверная колонка)
- **Unsafe SQL**: 0 (Ассистент не пытался выполнить удаление или модификацию данных).
- Парсинг JSON прошёл успешно для 100% ответов (все 14 примеров).

## 4. QLoRA Smoke Test
Выполнен тренировочный процесс SFT на 5 шагах для проверки backpropagation:
- **Адаптер**: Сохранён в `adapters/smoke_test/final`.
- **Trainable params**: 29,933,568 (0.96% от всех 3,115,872,256).
- Ошибок OOM не возникло, VRAM очистилась корректно после теста.

## Вывод
Базовая модель (Qwen 2.5 3B) "из коробки" справляется с половиной задач (50% Execution Match), однако испытывает сложности с генерацией идеально точного (Exact Match) SQL по строгим форматам таблиц.
Технологический пайплайн (NF4 QLoRA) доказал работоспособность и полностью готов к полноценной тренировке в Фазе 4.
