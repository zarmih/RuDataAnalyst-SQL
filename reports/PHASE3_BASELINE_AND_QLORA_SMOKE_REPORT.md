# Отчет по фазе 3: Базовый инференс и проверка QLoRA (Smoke Test)

## 1. Анализ модели
- **Основная модель**: `Qwen/Qwen3-4B`
- **Лицензия**: `Apache-2.0` (Коммерческое использование разрешено)
- **Размер загрузки (кэш)**: ~7.6 ГБ (safetensors, без GGUF).

**ЗАМЕЧАНИЕ О ЛИЦЕНЗИРОВАНИИ**:
Первоначально выбранная модель `Qwen/Qwen2.5-3B-Instruct` распространяется по лицензии `qwen-research`, которая разрешает использование деривативов ТОЛЬКО в НЕКОММЕРЧЕСКИХ целях. В связи с этим она была понижена до статуса legacy non-commercial baseline. Для обеспечения возможности публикации адаптера с пермиссивной лицензией, основной моделью выбрана `Qwen/Qwen3-4B` (Apache-2.0). 

**ОТКЛЮЧЕНИЕ THINKING-РЕЖИМА**:
Во время первичного диагностического прогона было выявлено, что модель Qwen3-4B при базовой генерации использует разметку `<think>`, что приводило к разрыву строгого JSON-формата по лимиту токенов (100% Parsed: False), снижению Execution Accuracy до 7.14% и увеличению latency до ~18 секунд на пример.
Для финального замера baseline (а также для smoke-тестирования) режим thinking был явно отключен через `tokenizer.apply_chat_template(..., enable_thinking=False)`, что полностью восстановило стабильность генерации (100% Parsed: True) и вернуло latency к ожидаемым 7-10 секундам.

## 2. NF4 Check & GPU Metrics
Проверка на NVIDIA GeForce RTX 5070 Ti (16 ГБ) показала, что квантование bitsandbytes **NF4** (4-bit) с compute dtype **bfloat16** стабильно работает на архитектуре Qwen3-4B.
- Пиковое потребление VRAM при инференсе: **2.65 GB**.

## 3. Baseline Results (Zero-shot, enable_thinking=False)
Инференс `test` выборки (14 примеров):
- **Exact Match Accuracy**: 14.29%
- **Execution Match Accuracy**: 50.00%
- **Unsafe SQL**: 0 (Ассистент не пытался выполнить опасные команды).
- Парсинг JSON прошёл успешно для 100% ответов (все 14 примеров).

## 4. QLoRA Smoke Test
Выполнен тренировочный процесс SFT на 5 шагах для проверки backpropagation:
- **Адаптер**: Сохранён в `adapters/smoke_test/final`.
- Ошибок OOM не возникло, VRAM очистилась корректно после теста. Тренировочный pipeline `trl/peft` полностью совместим с Qwen3.

## Вывод
Базовая модель `Qwen/Qwen3-4B` "из коробки" (в режиме strict-JSON) показывает 50% Execution Match и 14.29% Exact Match, что сопоставимо с метриками Qwen2.5. Технологический пайплайн (NF4 QLoRA) доказал работоспособность и готов к полноценной тренировке в Фазе 4, с гарантией лицензионной чистоты (Apache-2.0).
