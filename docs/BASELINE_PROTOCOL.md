# Протокол бейзлайн-инференса (Baseline Protocol)

## 1. Цель
Зафиксировать результаты работы базовой LLM до начала fine-tuning (QLoRA) на валидационных/тестовых данных, чтобы в дальнейшем корректно измерять дельту (прирост) метрик. 

## 2. Модель и конфигурация
- **Модель**: Qwen/Qwen3-4B
- **Загрузка**: 4-bit (bitsandbytes NF4) на GPU RTX 5070 Ti 16 GB.
- **Dtype вычислений**: bfloat16.

## 3. Выборка
- Тестовая выборка (test split): 14 запросов к базе 'support' (содержит SQL Schema и Вопросы на русском).

## 4. Промпт
- **Системный промпт**: "Ты SQL-ассистент. Отвечай только в формате JSON: {\"sql\": \"...\", \"explanation_ru\": \"...\", \"assumptions\": [], \"confidence\": \"high|medium|low\"}."
- **Пользовательский промпт**: Схема БД + Вопрос.

## 5. Метрики для отслеживания
- **Exact Match (EM)**: Доля полностью совпадающих с Gold SQL запросов.
- **Execution Match (EX)**: Доля запросов, которые при исполнении на SQLite-базе возвращают тот же результат, что и Gold SQL.
- **JSON Validity**: Форматирование и безопасность (нет деструктивных запросов типа DROP, UPDATE).
- **GPU Metrics**: Peak VRAM, среднее время генерации на 1 пример (latency).
